Getting Started

Everything you need to access pasteur for the first time — SSH setup, environment orientation, and basic verification steps.

New Users SSH

Step 1 — Get an Account

Access to pasteur requires an account provisioned by a cluster administrator. Contact Dr. Garibay or the CASL lab admins to request access. Provide your preferred username and university affiliation.

ℹ
Username policy Usernames, once created, cannot be reused if deleted. Choose carefully. Typically, your username matches your university NetID.

Step 2 — Connect via SSH

All user access is through the login node pasteur-login. You do not SSH directly to the compute node.

$ ssh your-username@pasteur-login.your-university.edu
Welcome to pasteur-login (Ubuntu 24.04 LTS)
CASL HPC Cluster — Login Node
Do not run compute jobs on this node. Submit jobs via Slurm.bash

Replace your-username with your cluster account name and pasteur-login.your-university.edu with the hostname provided by your admin.

SSH Key Setup (Recommended)

Password authentication works but key-based authentication is faster and more secure. Generate a key pair on your local machine if you don't already have one:

# On your LOCAL machine
$ ssh-keygen -t ed25519 -C "your-email@university.edu"
$ ssh-copy-id your-username@pasteur-login.your-university.edulocal

After this, SSH will authenticate automatically using your key.

Windows Users (MobaXterm)

If you're using MobaXterm:

  1. Create a new SSH session with the login node hostname
  2. Set your username in the session settings
  3. Use MobaXterm's built-in key manager to load your private key (optional but recommended)
  4. Save the session for future use

Step 3 — Explore Your Environment

Once logged in to pasteur-login, your home directory lives on the NFS-shared /data filesystem:

$ pwd
/data/home/your-username

$ ls ~
  (your home directory — 5 GB quota)

$ df -h /data/home
Filesystem      Size  Used Avail Use% Mounted on
pasteur:/data/home   14T  ...  ...  ...  /data/homepasteur-login

Your storage spans three areas. See the Storage Layout page for details on each:

  • /data/home/your-username — personal home directory
  • /data/scratch/your-username — fast temporary workspace
  • /data/shared/ — shared project data and containers

Step 4 — Verify Slurm Access

Check that you can see the Slurm cluster and your account:

# Check cluster status
$ sinfo
PARTITION  AVAIL  TIMELIMIT  NODES  STATE  NODELIST
debug*        up   1:00:00      1   idle   pasteur
standard      up  2-00:00:00   1   idle   pasteur
extended      up  7-00:00:00   1   idle   pasteur

# Check your account limits
$ sacctmgr show assoc user=$(whoami) format=user,partition,qos,maxgres -p
User|Partition|QOS|MaxGRES|
your-username||normal||pasteur-login

Step 5 — Run a Test Job

Submit a simple test job to verify your setup is working end-to-end:

# Submit an interactive test job
$ srun --partition=debug --time=00:05:00 --pty bash

# Inside the job — you are now on the compute node
$ hostname
pasteur

$ echo "Hello from the compute node!"
Hello from the compute node!

$ exit   # exit the interactive sessionpasteur-login

Or submit a batch job:

# Create a simple test script
$ cat > test_job.sh << 'EOF'
#!/bin/bash
#SBATCH --job-name=test
#SBATCH --partition=debug
#SBATCH --time=00:02:00
#SBATCH --output=test_%j.out

echo "Job ID: $SLURM_JOB_ID"
echo "Running on: $(hostname)"
echo "Date: $(date)"
EOF

# Submit it
$ sbatch test_job.sh
Submitted batch job 42

# Check its status
$ squeue -u $(whoami)
  JOBID PARTITION     NAME     USER ST       TIME  NODES NODELIST
     42     debug     test username  R       0:03      1 pasteur

# View the output once it completes
$ cat test_42.out
Job ID: 42
Running on: pasteur
Date: Thu Mar 26 12:34:56 UTC 2026pasteur-login
