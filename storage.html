Storage Layout

All user data lives under /data — a 14 TB drive on the compute node, exported via NFS and mounted on pasteur-login. Understand where to put your files before you start.

14 TB total NFS shared

Directory Structure

/data/ ← 14 TB, NFS root (on pasteur) ├── home/ NFS → /data/home on pasteur-login │ └── <username>/ ← your home directory (5 GB) ├── scratch/ NFS → /data/scratch on pasteur-login │ └── <username>/ ← fast temp workspace (part of 70 GB quota) ├── apps/ NFS → /data/apps on pasteur-login │ ├── casl/common/ ← shared conda envs (managed by admins) │ └── <username>/ ← per-user app installs (conda envs etc.) └── shared/ NFS → /data/shared on pasteur-login ├── containers/ ← shared Apptainer images (.sif) └── projects/ ← collaborative research data

Quotas

Each regular lab user has a combined quota of 70 GB across their home, scratch, and apps directories. This is enforced at the filesystem level.

User Type Quota Scope Notes
Lab user 70 GB Combined per user Across home + scratch + apps/<user>
Admin (casl-maintainer) 100 GB group Group quota arachchige, igaribay
Shared dirs Unlimited Shared project storage /data/shared/, /data/apps/casl/

Home Directory — /data/home/<username>

Your personal home directory. This is where your shell config files, scripts, and important outputs should live. It is accessible from both pasteur-login and the compute node.

  • Quota: 5 GB (hard limit)
  • Backed up: No — store important data here but be aware there is no automated backup
  • Accessible from: pasteur-login and pasteur (compute)
  • Good for: Shell configs, job scripts, small outputs, code repositories
⚠
5 GB is not much Keep large datasets and model checkpoints in scratch or shared/projects, not in your home directory. Your home directory filling up will cause login failures.

Scratch — /data/scratch/<username>

High-throughput temporary workspace intended for active job I/O. Use scratch for data that a running job reads and writes — it counts against your 70 GB combined quota.

  • Quota: Counts toward your 70 GB combined limit
  • Backed up: No — not backed up at all
  • Accessible from: pasteur-login and pasteur (compute)
  • Good for: Intermediate job outputs, large input datasets during an active experiment, model training checkpoints in progress
✕
Scratch is not permanent storage Files in scratch may be purged by admins if space is needed. Move important results to your home directory or shared project storage as soon as a job completes. 
# Use your scratch directory in job scripts
#!/bin/bash
#SBATCH --job-name=my_experiment

SCRATCH=/data/scratch/$USER
mkdir -p $SCRATCH/my_experiment_${SLURM_JOB_ID}

# Copy input data to scratch
cp /data/shared/projects/myproject/input.tar.gz $SCRATCH/

# ... run your job ...

# Copy important results back
cp -r $SCRATCH/results/ /data/home/$USER/completed/bash

Apps — /data/apps/

Software installations — primarily conda environments. There are two sub-areas:

  • /data/apps/casl/common/ — shared environments managed by admins, available to all users
  • /data/apps/<username>/ — personal environments you install yourself using mamba
# Create a personal conda environment in apps
$ mamba create --prefix /data/apps/$USER/my-env python=3.11

# Activate it
$ conda activate /data/apps/$USER/my-env

# Or use it in a Slurm job script
source /opt/miniforge3/etc/profile.d/conda.sh
conda activate /data/apps/$USER/my-envbash

Shared — /data/shared/

Collaborative storage for the whole lab. Writable by members of the casl-maintainer group; readable by all users.

  • /data/shared/containers/ — shared Apptainer .sif images (see Software Stack)
  • /data/shared/projects/ — collaborative research datasets
