Storage Layout

Storage lives on the compute node (pasteur) and is NFS-mounted on pasteur-login. Your paths depend on which group you belong to — your admin will tell you your group name.

14 TB total NFS shared

Directory Structure

The following shows the paths as seen from pasteur-login (what you see when you log in):

/mnt/datahome/ ← home directories └── <username>/ ← your home directory /scratch/ └── <group>/ └── <username>/ ← fast temp workspace (not backed up) /apps/ └── <group>/ └── <username>/ ← your conda envs and personal software /shared/ ├── archives/ ← shared archived data └── projects/ ← shared project data
ℹ
Your group name The <group> in paths like /scratch/<group>/<username> is assigned when your account is created. It may be a lab name (e.g. casl) or a project name. Your admin will tell you your group.

Quotas

Each regular lab user has a combined quota of 70 GB across their home, scratch, and apps directories. This is enforced at the filesystem level.

User Type Quota Scope Notes
Lab user 70 GB Combined per user Across home + scratch + apps/<user>
Shared dirs Unlimited Shared project storage /shared/ — not counted against user quotas

Check your current usage:

$ quota -s

pasteur:/data   12G    70G    75Gpasteur-login

Home Directory — /mnt/datahome/<username>

Your personal home directory. This is where your shell config files, scripts, and important outputs should live. It is accessible from both pasteur-login and the compute node.

  • Quota: Counts toward your 70 GB combined quota (shared with scratch and apps/<user>)
  • Backed up: No — store important data here but be aware there is no automated backup
  • Accessible from: pasteur-login and pasteur (compute)
  • Good for: Shell configs, job scripts, small outputs, code repositories
⚠
Your quota is shared across home, scratch, and apps Large datasets or model checkpoints stored in your home directory eat into the same 70 GB pool as everything else. Keep bulky files in scratch or shared/projects — a full quota causes login failures.

Scratch — /scratch/<group>/<username>

High-throughput temporary workspace intended for active job I/O. Use scratch for data that a running job reads and writes — it counts against your 70 GB combined quota.

  • Quota: Counts toward your 70 GB combined limit
  • Backed up: No — not backed up at all
  • Accessible from: pasteur-login and pasteur (compute)
  • Good for: Intermediate job outputs, large input datasets during an active experiment, model training checkpoints in progress
✕
Scratch is not permanent storage Files in scratch may be purged by admins if space is needed. Move important results to your home directory or shared project storage as soon as a job completes. 
#!/bin/bash
#SBATCH --job-name=my_experiment

# Paths inside a Slurm job (compute node)
GROUP=$(id -gn)
SCRATCH=/data/scratch/$GROUP/$USER
mkdir -p $SCRATCH/run_${SLURM_JOB_ID}

# Copy input data to scratch
cp /data/shared/projects/myproject/input.tar.gz $SCRATCH/run_${SLURM_JOB_ID}/

# ... run your job ...

# Copy important results back to home when done
cp -r $SCRATCH/run_${SLURM_JOB_ID}/results/ $HOME/completed/job.sh

Apps — /apps/<group>/

Software installations — primarily conda environments. Your personal area is at /apps/<group>/<username>/ on pasteur-login.

# Create a personal conda environment (on pasteur-login)
$ mamba create --prefix /apps/$(id -gn)/$USER/my-env python=3.11

# Activate it interactively
$ conda activate /apps/$(id -gn)/$USER/my-envpasteur-login
# In a Slurm job script, paths are under /data/apps/ on the compute node
GROUP=$(id -gn)

source /opt/miniforge3/etc/profile.d/conda.sh
conda activate /data/apps/$GROUP/$USER/my-envjob.sh

Shared — /shared/

Read-only shared storage. Access is granted per project by an admin. In Slurm job scripts use /data/shared/.

  • /shared/archives/ — shared archived data
  • /shared/projects/ — shared project datasets

Best Practices

Use caseRecommended location
Job scripts & code$HOME (/mnt/datahome/<user>)
Input datasets for a running job/scratch/<group>/<user> (login) · /data/scratch/<group>/<user> (job)
Completed experiment results$HOME
Shared project data/shared/projects/
Archived data/shared/archives/
Conda environments (personal)/apps/<group>/<user>/ (login) · /data/apps/<group>/<user>/ (job)
